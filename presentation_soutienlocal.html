Slide 3
Introduction
Contexte & Problématique

📌 Contexte

Le soutien scolaire au Maroc reste largement informel et décentralisé. Les élèves peinent à trouver des tuteurs qualifiés dans leur proximité, tandis que les tuteurs manquent de visibilité et d'outils de gestion de leurs cours.

❓ Problématique

Comment digitaliser la mise en relation élève–tuteur en offrant une plateforme web moderne, sécurisée et intuitive permettant la recherche, la réservation et le suivi des séances de soutien scolaire ?

⚡ Défis identifiés
  • Absence de plateforme dédiée au soutien scolaire local
  • Difficulté pour les élèves de comparer les tuteurs
  • Processus de réservation non structuré
  • Manque de suivi et de traçabilité des séances
  • Besoin d'un panel d'administration centralisé

Slide 17
Retour d'expérience
Difficultés Rencontrées

⚠️ Configuration CORS

Communication entre le frontend React (port 3000) et l'API Laravel (port 8000) bloquée par les politiques CORS.

✅ Configuration du middleware CORS dans Laravel + proxy React.

⚠️ Gestion de l'état global

Synchronisation complexe de l'état utilisateur entre les composants avec l'authentification.

✅ Implémentation de React Context API (AuthContext) pour centraliser l'état.

⚠️ Responsive Design complexe

Adaptation du dashboard admin et des tableaux de données sur écrans mobiles.

✅ Approche Mobile-First + composants adaptatifs + masquage conditionnel.

⚠️ Sécurité des routes

Protection des routes admin et gestion des tokens JWT expirés.

✅ Middleware Auth + intercepteurs Axios pour le refresh automatique.
